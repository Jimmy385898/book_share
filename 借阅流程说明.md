# 借阅流程详细说明

## 完整的借阅流程

### 1. 图书发布阶段
```
用户发布图书 → 状态：待审核(0)
     ↓
管理员审核通过 → 状态：可借阅(1)
```

### 2. 借阅申请阶段
```
借阅者浏览图书 → 点击"申请借阅"
     ↓
填写借阅信息：
  - 借阅天数
  - 交接方式（面对面/代收点/快递）
  - 备注说明
     ↓
提交申请 → 借阅记录状态：待确认(0)
```

### 3. 出借方确认阶段
```
出借方收到申请通知
     ↓
查看借阅者信息和信用分
     ↓
选择操作：
  - 同意借阅 → 状态：已确认(1)
  - 拒绝借阅 → 状态：已拒绝(5)
     ↓
同意后自动生成电子协议
图书状态变更为：已借出(2)
```

### 4. 电子协议生成
```
协议内容包括：
  - 协议编号
  - 双方信息（姓名、联系方式）
  - 图书信息（书名、作者、ISBN）
  - 借阅条款（期限、交接方式、地址）
  - 双方责任和承诺
  - 违约责任
  - 电子签名和时间戳
```

### 5. 图书交接阶段
```
出借方确认发货 → 状态：借阅中(2)
     ↓
交接方式：
  - 面对面：约定时间地点交接
  - 代收点：放置到社区代收点
  - 快递：填写快递单号
```

### 6. 借阅中状态追踪
```
系统自动监控：
  - 借阅天数倒计时
  - 逾期检测（每日定时任务）
  - 续借次数统计（最多2次）
     ↓
借阅者可以：
  - 查看电子协议
  - 申请续借（延长归还期限）
  - 提前归还
     ↓
如果逾期：
  - 标记为逾期状态
  - 每天扣除信用分1分
  - 影响后续借阅权限
```

### 7. 续借流程
```
借阅者申请续借
     ↓
检查条件：
  - 当前状态必须是"借阅中"
  - 续借次数 < 2次
  - 图书允许续借
  - 未逾期
     ↓
续借成功 → 延长归还期限
续借次数 +1
```

### 8. 归还阶段
```
借阅者确认归还 → 状态：已归还(3)
     ↓
系统检查：
  - 是否逾期
  - 计算实际借阅天数
     ↓
如果逾期：
  - 标记逾期
  - 扣除信用分
     ↓
图书状态恢复为：可借阅(1)
```

### 9. 取消/拒绝流程
```
借阅者取消申请（仅待确认状态）
  → 状态：已取消(4)
     
出借方拒绝申请
  → 状态：已拒绝(5)
```

## 状态流转图

```
待确认(0) ──同意──→ 已确认(1) ──发货──→ 借阅中(2) ──申请归还──→ 待确认归还(6) ──确认──→ 已归还(3)
    │                                    │
    │                                    │
  取消/拒绝                            续借(延长期限)
    │                                    │
    ↓                                    ↓
已取消(4)                          借阅中(2)
已拒绝(5)                          (续借次数+1)
```

## 图书状态说明

- **待审核(0)**：用户刚发布，等待管理员审核
- **可借阅(1)**：审核通过，可以被借阅
- **已借出(2)**：正在被借阅中
- **下架(3)**：管理员下架或用户删除

## 借阅记录状态说明

- **待确认(0)**：借阅者已申请，等待出借方确认
- **已确认(1)**：出借方同意，等待发货
- **借阅中(2)**：图书已交接，正在借阅
- **已归还(3)**：借阅完成，图书已归还
- **已取消(4)**：借阅者取消申请
- **已拒绝(5)**：出借方拒绝申请
- **待确认归还(6)**：借阅者申请归还，等待出借方确认

## 关键功能点

### 1. 双向确认机制
- **借阅确认**：借阅者发起申请 + 出借者确认同意
- **归还确认**：借阅者申请归还 + 出借者确认归还
- 双方都可以查看协议
- 保证流程的公平性和规范性

### 2. 电子协议
- 自动生成正式协议
- 包含完整的借阅条款
- 双方电子签名
- 可随时查看

### 3. 状态追踪
- 实时显示借阅状态
- 逾期自动标记
- 信用分自动扣除
- 借阅历史记录

### 4. 续借管理
- 最多续借2次
- 每次最多30天
- 需要图书允许续借
- 不能逾期续借

### 5. 提前归还
- 借阅者可随时归还
- 不影响信用分
- 图书立即可借

### 6. 逾期处理
- 自动检测逾期
- 每天扣1分信用分
- 最多扣30分
- 影响借阅权限

## 数据库字段说明

### borrow_record 表关键字段

```sql
apply_time          -- 申请时间（借阅者提交申请）
confirm_time        -- 确认时间（出借方同意）
delivery_time       -- 发货时间（出借方确认发货）
return_time         -- 应还时间（计算的归还截止日期）
actual_return_time  -- 实际归还时间（借阅者申请归还时记录）
status              -- 当前状态（0-6共7个状态）
is_overdue          -- 是否逾期
renew_count         -- 续借次数
```

## 使用示例

### 借阅者操作流程
1. 浏览图书 → 查看详情
2. 点击"申请借阅" → 填写信息
3. 等待出借方确认
4. 查看电子协议
5. 按约定方式取书
6. 借阅期间可续借
7. **归还图书 → 点击"申请归还"**
8. **等待出借方确认归还**

### 出借方操作流程
1. 收到借阅申请通知
2. 查看借阅者信用分
3. 同意或拒绝申请
4. 查看电子协议
5. 确认发货
6. **收到归还申请**
7. **验证图书完好**
8. **点击"确认归还"完成流程**

### 管理员操作
1. 审核新发布的图书
2. 查看所有借阅记录
3. 处理纠纷和投诉
4. 监控逾期情况
5. 管理用户信用分
