# 快速启动指南

## 环境要求

- JDK 1.8+
- Maven 3.6+
- MySQL 8.0+
- Redis 5.0+
- Node.js 16+

## 启动步骤

### 1. 启动 MySQL

确保 MySQL 服务已启动，然后执行数据库初始化：

```bash
# 登录 MySQL
mysql -u root -pjimmy

# 执行初始化脚本
source D:/课程资料/毕设参考/zjp/database.sql
```

### 2. 启动 Redis

```bash
# Windows 下启动 Redis
redis-server.exe

# 或者如果已安装为服务
net start Redis
```

### 3. 启动后端服务

```bash
# 进入后端目录
cd D:/课程资料/毕设参考/zjp/backend

# 使用 Maven 编译打包
mvn clean package -DskipTests

# 运行
java -jar target/community-book-share-1.0.0.jar

# 或者在 IDE 中直接运行 BookShareApplication.java
```

后端服务启动成功后，访问 http://localhost:8080

### 4. 启动用户端前端

```bash
# 进入用户端目录
cd D:/课程资料/毕设参考/zjp/front/user

# 安装依赖（首次运行）
npm install

# 启动开发服务器
npm run dev
```

用户端启动成功后，访问 http://localhost:3000

### 5. 启动后台管理端前端

```bash
# 进入后台管理端目录
cd D:/课程资料/毕设参考/zjp/front/admin

# 安装依赖（首次运行）
npm install

# 启动开发服务器
npm run dev
```

后台管理端启动成功后，访问 http://localhost:3001

## 常见问题

### 1. 端口被占用

如果端口被占用，可以修改配置文件：

- 后端端口：修改 `backend/src/main/resources/application.yml` 中的 `server.port`
- 用户端端口：修改 `front/user/vite.config.js` 中的 `server.port`
- 管理端端口：修改 `front/admin/vite.config.js` 中的 `server.port`

### 2. 数据库连接失败

检查 `application.yml` 中的数据库配置：
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/book_share?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false
    username: root
    password: jimmy
```

### 3. Redis 连接失败

确保 Redis 服务已启动，检查 `application.yml` 中的 Redis 配置：
```yaml
spring:
  redis:
    host: localhost
    port: 6379
```

### 4. 文件上传失败

确保上传目录存在且有写入权限：
```yaml
file:
  upload:
    path: D:/bookshare/uploads/
```

系统会自动创建该目录，如果失败请手动创建。

### 5. 前端无法访问后端 API

检查前端代理配置是否正确，确保后端服务已启动。

### 重要提示

1. **图书状态管理**：
   - 新发布的图书状态为"待审核(0)"
   - 只有审核通过(状态1)的图书才能被借阅
   - 借阅中的图书状态为"已借出(2)"
   - 归还后恢复为"可借阅(1)"

2. **借阅记录状态**：
   - 0-待确认：等待出借方确认
   - 1-已确认：出借方同意，等待发货
   - 2-借阅中：正在借阅
   - 3-已归还：借阅完成
   - 4-已取消：借阅者取消
   - 5-已拒绝：出借方拒绝

3. **电子协议**：
   - 只有状态>=1的借阅记录才能查看协议
   - 协议包含完整的借阅条款和双方责任
   - 双方都可以查看协议

4. **续借限制**：
   - 最多续借2次
   - 每次最多30天
   - 必须在借阅中状态
   - 图书必须允许续借
   - 不能逾期续借

5. **逾期处理**：
   - 系统自动检测逾期
   - 每天扣除1分信用分
   - 最多扣除30分
   - 影响后续借阅权限

## 生产部署

### 后端部署

```bash
# 打包
mvn clean package -DskipTests

# 运行（后台运行）
nohup java -jar target/community-book-share-1.0.0.jar > app.log 2>&1 &
```

### 前端部署

```bash
# 用户端构建
cd front/user
npm run build

# 管理端构建
cd front/admin
npm run build

# 将 dist 目录部署到 Nginx 或其他 Web 服务器
```

### Nginx 配置示例

```nginx
# 用户端
server {
    listen 80;
    server_name your-domain.com;
    
    location / {
        root /path/to/user/dist;
        try_files $uri $uri/ /index.html;
    }
    
    location /api {
        proxy_pass http://localhost:8080;
    }
    
    location /uploads {
        proxy_pass http://localhost:8080;
    }
}

# 管理端
server {
    listen 80;
    server_name admin.your-domain.com;
    
    location / {
        root /path/to/admin/dist;
        try_files $uri $uri/ /index.html;
    }
    
    location /api {
        proxy_pass http://localhost:8080;
    }
}
```
